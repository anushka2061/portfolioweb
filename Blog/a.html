Elijah's CTF

In this project, I reverse-engineered a Golang binary, which was a part of the Elijah's CTF challenge. The challenge offered 300 points for a successful exploit.

Challenge Overview

The task involved reverse-engineering a Golang binary, which was intentionally obfuscated to prevent easy analysis. The goal was to find vulnerabilities in the code that would allow us to extract a hidden flag.

Steps Taken

1. **Initial Analysis**: I began by running the binary and observing its behavior. I noticed a series of cryptographic functions that seemed out of place.
2. **Disassembly**: Using tools like Ghidra, I disassembled the binary and examined the instructions to identify potential weaknesses.
3. **Debugging**: I used a debugger to step through the code, which revealed a vulnerability in the random number generation mechanism, leading to the extraction of the flag.

Challenges Faced

One of the major challenges was understanding the custom obfuscation techniques used in the Golang binary. The code was intentionally convoluted, requiring advanced reverse-engineering techniques to untangle it.

Solution

By carefully analyzing the disassembled code, I identified a critical bug in the random number generation. Exploiting this vulnerability allowed me to predict the key used to generate the flag, which led to a successful capture.

Key Learnings

- Understanding Golang's memory management was crucial for disassembling the binary.
- Reverse-engineering obfuscated code requires patience and careful attention to detail.
- Debugging and stepping through the code is often the key to discovering hidden flaws.
